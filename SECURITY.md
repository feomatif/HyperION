# Security Policy

## Reporting a vulnerability

Please do not open a public GitHub issue for a suspected vulnerability.

Send a private report through the [official HyperION contact page](https://www.hyperiondaw.com/contact) and clearly identify it as a security report. Include:

- The affected HyperION edition and version
- A clear description of the issue and its impact
- Reproduction steps or a minimal proof of concept
- Any relevant logs with secrets and personal information removed

Please allow a reasonable amount of time for investigation before sharing details publicly. We will confirm receipt, assess the report, and coordinate next steps when the issue can be reproduced.

## Scope

Security reports should concern HyperION application builds or official distribution channels. General bugs and feature requests belong in the public [issue tracker](https://github.com/hyperiondaw/HyperION/issues).
