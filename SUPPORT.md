# HyperION Support

Thanks for using HyperION. This repository is the best place for reproducible software bugs and focused feature requests.

## Before opening an issue

1. Confirm that you are using the latest build from the [official download page](https://www.hyperiondaw.com/download) or HyperION Setup.
2. Restart HyperION and check whether the issue can be reproduced in a new project.
3. Note the HyperION edition and version, Windows version, audio interface, sample rate, buffer size, and relevant plugin version.
4. Remove personal information, license details, private project files, and copyrighted audio you cannot share.

## Bug reports

Use the [bug report form](https://github.com/hyperiondaw/HyperION/issues/new?template=bug_report.yml) and include concise reproduction steps. Sanitized logs, crash dumps, plug-in scan reports, and screenshots are useful when they do not contain sensitive data.

## Feature requests

Use the [feature request form](https://github.com/hyperiondaw/HyperION/issues/new?template=feature_request.yml). Describe the production problem first, then the workflow you would like HyperION to support.

## Purchase and download help

For receipts, download access, billing, licenses, or account-specific questions, use the [official contact page](https://www.hyperiondaw.com/contact). If you purchased through a supported partner, you can [claim your license key](https://www.hyperiondaw.com/claim-license-key). Do not post purchase or license information in a public GitHub issue.

Common setup and licensing answers are available in the [HyperION FAQ](https://www.hyperiondaw.com/faq).

## Security

Do not disclose a suspected security vulnerability in a public issue. Follow the private reporting guidance in [SECURITY.md](SECURITY.md).
